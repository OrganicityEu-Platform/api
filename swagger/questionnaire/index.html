Questionnaires

This API specifices access to the questionnaire of the scenario tool.

Default request content-types: application/json
Default response content-types: application/json
Schemes: http

Summary

Path Operation Description
/questionnaire GET

Returns the currently active questionnaire, or a list of all questionnaires if requested.
PATCH

Updates the questionnaire. Only used for cosmetic updates.

Paths

Returns the currently active questionnaire, or a list of all questionnaires if requested.

GET /questionnaire

If no version is supplied, this will return the current questionnaire. Usually, questionnaires are versioned in increasing numbers, so if the current questionnaire has version number 7, the previous versions can be requested by passing version as 6, 5, and so on.
version

The requested version, or 'all' if all versions should be returned.

 query string (Either 'all' or a version number.)

Uses default content-types: application/json

200 OK
  • If successful, this returns the current questionnaire if no version was specificed, or the questionnaire of the specified version. - If all versions are requested, this returns a list of questionnaire objects.
Questionnaire
404 Not Found

Returned if no questionnaire with the given version number exists.

Schema definitions

Question: object

A single question belonging to a questionnaire.

{
    "tech": true,
    "text": "Are existing technologies capable of handling this scenario?",
    "values": [
        {
            "value": "Yes",
            "weight": 1
        },
        {
            "value": "No",
            "weight": -1
        }
    ]
}
tech: boolean

true if this question is a question is a technical question

text: string

The question description that is shown to the user

value: object[]

The possible answers the user can choose from

object
value: string

A textual description of the answer that is shown to the user

weight: integer

The weight that this answer applies to the overall rating. Higher values will have a larger effect.

Questionnaire: object

A single questionnaire.

{
    "description": "Second revision of initial questionnaire, fixed typos.",
    "explanation": "Please rate the scenario provided from your personal perspective.",
    "questions": [
        {
            "tech": false,
            "text": "Does this idea apply to your daily life?",
            "values": [
                {
                    "value": "This does not affect me.",
                    "weight": -1
                },
                {
                    "value": "Don't really know, maybe?",
                    "weight": 0
                },
                {
                    "value": "Yes, I can use that.",
                    "weight": 1
                }
            ]
        },
        {
            "tech": true,
            "text": "Is this scenario plausible?",
            "values": [
                {
                    "value": "No",
                    "weight": -1
                },
                {
                    "value": "Yes",
                    "weight": 1
                }
            ]
        }
    ],
    "version": 2
}
version: integer

The version name of this questionnaire. The version is assigned by the server, where it is incremented by one for each new version posted.

author: string (uuid)

The UUID of the user that created this version of the questionnaire.

description: string

A description of this questionnaire for internal use. This text is not displayed to the front end users.

explanation: string

An explanatory text that is shown to users at the top of the questionnaire.

questions: object[]

The list of questions this questionnaire asks. These might be classified as technical or non-technical, and usually only the subset matching the user's choice should be shown as choices.
